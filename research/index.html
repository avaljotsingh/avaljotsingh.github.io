Research

My work is organized around two tightly connected directions: using formal methods to make AI systems trustworthy, and using AI to automate and scale formal methods.

Formal Methods for AI

I build languages, verifiers, and compilers so that neural network analyses (e.g. certifiers for robustness) can be specified declaratively, checked for soundness, and executed efficiently. The following projects form a coherent pipeline: specification → soundness verification → compiled execution.

ConstraintFlow

A declarative DSL for writing neural network certifiers based on abstract interpretation. Specifications are concise and amenable to both automated verification and compilation.

SAS

ProveSound

Automatically verifies that certifiers written in ConstraintFlow are sound (over-approximate). This ensures that any robustness certificate produced by the certifier is mathematically valid.

OOPSLA

Neuton

A tensor-based compiler and runtime for executing ConstraintFlow specifications on hardware. Enables efficient neuron-level analysis of neural networks.

arXiv (under submission)

AI for Formal Methods

I use AI and synthesis to automate formal methods and program analysis: generating new certifiers, generating program optimizations, and assisting theorem proving.

SAIL

Synthesizes new neural network certifiers automatically. It uses the ConstraintFlow DSL, its verifier (ProveSound), and its compiler (Neuton) to produce sound abstract interpreters optimized for a given cost objective.

arXiv

RuleFlow

Generates reusable data-processing and Pandas optimizations using LLMs combined with compiler techniques. Discovered optimizations are turned into rewrite rules and applied automatically.

arXiv

Theorem Proving

Using AI and large language models to assist or automate theorem proving.
